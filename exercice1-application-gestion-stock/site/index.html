Ce site présente un aperçu visuel des résultats d'exécution et un guide rapide. Il est prêt pour un déploiement sur Render en tant que site statique.

1) Résultats après exécution — Analyse des captures

1. Tests JUnit — tous passent

  • testListProduitsParCategorie
  • testProduitsCommandesEntreDeuxDates
  • testProduitsParCommande
  • testProduitsPrixSup100NamedQuery

Initialisation: 2 catégories, 4 produits, 2 commandes, 4 lignes de commande.

2. Tables MySQL créées

Les tables categorie, commande, produit, ligne_commande_produit sont bien créées et peuplées.

2) Lancer le projet (local)

  1. Prérequis: Java 17+, Maven 3.8+.
  2. Depuis la racine du module: mvn -q -DskipTests=false test
  3. Option: Exécuter la classe ma.projet.App depuis l'IDE.

Configurer la base via src/main/resources/application.properties si nécessaire.

3) Fonctionnalités testées

  • Lister les produits d’une catégorie donnée.
  • Lister les produits commandés entre deux dates.
  • Lister les produits d’une commande avec prix et quantités (jointure).
  • Requête nommée pour les produits dont le prix est > 100.

4) Déploiement sur Render (site statique)

  1. Forkez ou poussez ce dépôt sur votre propre compte Git.
  2. Sur Render, créez un nouveau Static Site.
  3. Sélectionnez le dépôt et la branche.
  4. Publish Directory: site
  5. Build Command: laissez vide, ou utilisez la configuration du fichier render.yaml fourni pour copier les images.

Ce projet inclut un fichier render.yaml qui définit un service static_site, publie le dossier site et copie automatiquement les captures pour qu'elles soient visibles en production.

Après déploiement, communiquez l'URL https://votre-site.onrender.com à votre professeur.
